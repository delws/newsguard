# NEWSGUARD

RAG-система валидации достоверности новостных постов относительно корпуса доверенных украинских СМИ (белый список ІМІ), с генерацией дайджестов.

**Ключевые принципы, зашитые в код и промпты:**

- `no_data` ≠ фейк. Эксклюзив или региональная новость может законно отсутствовать в корпусе. Судья обязан ставить `no_data` при отсутствии релевантных фрагментов и не додумывать.
- Новости живут часами: векторный поиск **всегда** фильтруется по времени (±48 ч от даты поста, `retrieval.window_hours`).
- Валидируются **утверждения** (claims), не «канал целиком». Рейтинг канала — статистика по истории claims.
- Эталонный корпус — не «истина», а консенсус доверенных источников.

Лицензии стека: только Apache 2.0 / MIT. Стоимость на этапе разработки — ноль (Groq free tier, локальные embeddings на CPU).

## Быстрый старт

```bash
# 1. БД (PostgreSQL 16 + pgvector)
docker compose up -d

# 2. Зависимости (Python 3.12, uv)
uv sync
# если планируете provider: anthropic
uv sync --extra anthropic

# 3. Ключи
cp .env.example .env
# впишите GROQ_API_KEY — бесплатно, без карты: https://console.groq.com

# 4. Применить схему БД
uv run python -m newsguard.db
```

Дальше (по мере готовности шагов):

```bash
uv run python -m newsguard.fetcher     # шаг 2: наполнить корпус эталонами
uv run python test_channel.py --source <tg_username|url> --days 3   # шаг 4
uv run python test_channel.py --gold   # прогон gold-источников
```

## Как добавить источник

Добавьте запись в `config.yaml` → `sources`:

```yaml
- name: Название
  kind: rss | telegram | web
  identifier: <URL фида | tg-хендл без @ | URL сайта>
  role: reference | user     # reference — эталон, user — проверяемый источник
  topic: [politics]
  trust_note: почему источник в списке
```

Для `telegram` указывайте хендл, подтверждённый на официальном сайте издания, — не угадывайте.

## Как читать вывод test_channel

Для каждого извлечённого claim печатается:

- **verdict** — `supported` (подтверждён корпусом) / `contradicted` (противоречит) / `no_data` (в корпусе нет релевантного — это НЕ признак фейка);
- **confidence** — уверенность судьи 0..1;
- найденные эталонные фрагменты со ссылками на первоисточники;
- **reasoning** — обоснование судьи.

## Как переключить провайдера судьи

В `config.yaml` → `llm.judge` меняете блок целиком (provider, model, base_url, api_key_env). Модель никогда не хардкодится в коде — только конфиг. Примеры для Google AI Studio и Anthropic закомментированы в конфиге. Отдельный блок `llm.judge_alt` позволяет сравнить двух судей бок о бок в `test_channel.py --gold`.

## Провайдеры инференса

| Провайдер | Free tier | Ориентировочные лимиты* | OpenAI-совместимый |
|---|---|---|---|
| Groq | да, без карты | ~30 RPM / ~1–14K RPD в зависимости от модели | да |
| Google AI Studio | да | ~10–30 RPM, дневные квоты по модели | да (`/v1beta/openai`) |
| OpenRouter | частично (`:free`-модели) | ~20 RPM, ~50–1000 req/день | да |

\* Лимиты меняются — сверяйте с актуальной документацией провайдера.

> **Free-тиры — ТОЛЬКО для разработки.** Условия бесплатных тиров не дают SLA, лимиты и модели меняются без предупреждения, а данные могут использоваться провайдером. Для продакшена: самохостинг Qwen (Apache 2.0) на GPU-VPS либо платный API с коммерческими условиями (при anthropic код автоматически использует prompt caching и Batch API; заполните `price_in_per_mtok`/`price_out_per_mtok` в конфиге — стоимость будет логироваться).

## Локальный офлайн-режим (опционально)

Полностью локальный судья через Ollama:

```yaml
judge:
  provider: ollama
  model: qwen3:4b
  base_url: http://localhost:11434/v1
  api_key_env: null
```

У Qwen3 обязательно **отключите thinking-режим** (в Ollama — `/set nothink`, либо суффикс `/no_think` в системном промпте): иначе модель тратит токены на рассуждения вместо JSON.

⚠️ Предупреждения: на CPU-ноутбуке генерация очень медленная (минуты на один claim), а качество судьи на 4B-модели заметно ниже, чем у 70B+ — вердикты годятся для отладки пайплайна, не для оценки источников.
